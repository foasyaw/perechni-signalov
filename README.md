# Перечни сигналов из схем подключения — v2.4

Приложение автоматизирует выпуск перечней входных/выходных сигналов АСУ ТП:
читает чертежи схем подключения (общие виды шкафов, DWG/DXF), извлекает все
каналы модулей ввода-вывода и заполняет фирменные Word-перечни с сохранением
рамок ГОСТ, штампов и содержания.

## Запуск

**Перечни_из_схем.exe** — автономный, один файл, ничего устанавливать не нужно.

Из исходников: `python perechni_gui.py` (нужны `pip install ezdxf python-docx openpyxl`).

## Страницы

### ◈ Дашборд
Сводка проекта: шкафы, каналы, занято/резерв %, замечания проверок. Таблица
**свободных каналов** (куда посадить новый сигнал) с фильтром по типу. Глобальный
поиск по всем шкафам. Кнопка **«Штампы пакета…»** — массовая замена дат/стадии
в колонтитулах docx (старое значение определяется автоматически, создаются .bak).

### ▣ Сборка перечней
Чертежи (порядок = порядок разделов, имя раздела правится двойным щелчком) →
шаблоны .docx (любой прежний перечень) → папка результата → **Собрать перечни**.
Галка «Обновить номера страниц через Word» проставляет номера в содержании
автоматически (Word запускается в фоне).

### ≣ Сигналы
Предпросмотр всех извлечённых каналов до сборки: фильтр по типу (AI/AO/DI/DO/WI/Резерв),
поиск, правка тега/описания/взрывозащиты двойным щелчком (правки попадают в перечни),
**экспорт сводки в Excel**, кнопка **«⌖ В AutoCAD»** — открывает чертёж и зумит прямо на выбранный канал.

### ⇄ Конвертер DWG → DXF
Пакетная конвертация. Движок ищется сам: **AutoCAD** (accoreconsole.exe, ~10 с/файл)
или **ODA File Converter** (бесплатный, opendesign.com). Результат можно сразу
добавить в сборку. DWG можно класть и прямо в сборку — сконвертируются на лету.

### Δ Сравнение версий
Сравнивает данные чертежей с прежними перечнями .docx: **добавлено / удалено /
изменено** по каждой позиции, с отчётом в Excel. Удобно при внесении изменений
в проект.

### ⚙ Настройки, ⓘ О программе
Путь конвертера (авто: AutoCAD → ODA), тёмная тема.

### Прочее
- **Проекты**: выпадающий список на «Сборке» — сохраняй наборы (чертежи+шаблоны+папка) и переключайся одним кликом.
- **Проверки** (кнопка на «Сигналах»): дубли позиций, «Резерв» с тегом, описание без тега, пропуски каналов, один тег с разными описаниями.
- **PDF**: галка на «Сборке» — рядом с .docx сохраняется .pdf (через Word в фоне).
- **Excel как источник**: в список чертежей можно добавить Excel-сводку (формат экспорта со страницы «Сигналы») — её разделы попадут в перечни. Цикл «чертежи → Excel → правка → перечни».
- **Пакетная сборка**: одна кнопка — перечни + Excel-сводка + спецификация + заготовка ПЗ; складывается в папку «Ревизия NN — дата» с манифестом (что из чего собрано, md5 чертежей, замечания).
- **Заготовка ПЗ (Word)**: сводка сигналов, состав ТС, ведомость КИП — на странице «Спецификация».
- **Drag & drop**: чертежи можно перетаскивать мышью прямо в списки.
- **Темы**: светлая и тёмная (Настройки → Оформление), тёплая палитра.

## Что берётся из чертежа

| Колонка перечня        | Источник |
|------------------------|----------|
| Позиция по проекту     | тег из описания канала («поз. PIT-1(23).») |
| Описание сигнала       | текст описания (без «поз. ...») |
| Уровень управления     | дискретные — «СК НО», аналоговые — пусто |
| Уровень сигнала        | аналоговые «4-20 мА», дискретные «=24В» |
| Взрывозащита           | **Exia**, если у модуля есть БИЗ, иначе **Exd** |
| Позиция в контроллере  | `модуль.канал` — например `2.1.1`, `2.19.1` |

Резервные каналы включаются строками «Резерв». Повторяющиеся позиции
подсвечиваются в журнале («ВНИМАНИЕ»).

## Ожидаемый стиль чертежа (фирменный)

- заголовки аналоговых модулей: текст `Модуль X.Y AI|AO|WI ...`;
- описания каналов: MTEXT на слое **Текст**, содержат `поз. <ТЕГ>.`;
- блоки искрозащиты: подписи `БИЗ-...` на слое **mark2**;
- дискретные каналы: метки `X.Y-KLDI<n>` / `X.Y-KLDO<n>` на слоях
  **Реле DI (1-KL)** / **Реле DO (3-KL)**.

## Об авторстве

Разработано в связке «инженер + ИИ» (Claude, Anthropic): постановка задачи,
предметная область, нормоконтроль и проверка результата — инженер;
код и разбор форматов DXF/DOCX — ИИ под контролем инженера.

Проверено на реальном проекте: 645 сигналов, результат
идентичен перечням, собранным вручную; сравнение «чертежи ↔ перечни»
даёт 0 расхождений.
